## Design Notes

### Neobrutalism Elements:
- **Bold, chunky buttons** with thick borders and shadows
- **High contrast colors** - primarily blue backgrounds with white/black text
- **Grid-based layouts** with clear geometric sections
- **Thick borders** around all major components
- **Minimal rounded corners** or completely square elements
- **Bold typography** with clear hierarchy

### Color Scheme:

- **Primary Blue**: #0b81ffff (for hero and CTA sections)
- **White/Light Gray**: #FFFFFF / #F8FAFC (for content areas)
- **Black**: #000000 (for text and borders)
- **Accent Colors**: Use sparingly for icons and highlights

### Typography:
- **Headers**: Bold, sans-serif fonts (Inter, Helvetica, or similar)
- **Body**: Medium weight for readability
- **Buttons**: Bold, uppercase or sentence case

### Interactive Elements:
- **Buttons**: Thick borders, shadow effects, hover states with color shifts
- **Cards**: Drop shadows, thick borders, slight hover animations
- **Navigation**: Clean, minimal with clear active states

### Responsive Considerations:
- Stack feature cards vertically on mobile
- Adjust hero text size for smaller screens
- Maintain button sizes for touch interaction
- Keep grid structure but allow for single-column layout

This layout captures the neobrutalism aesthetic while presenting Scribesent's core value proposition clearly and effectively.

**Source:** https://www.neobrutalism.dev  
**Library:** Neobrutalism Components by ekmas  
**Repository:** https://github.com/ekmas/neobrutalism-components



### UI/UX Testing Requirements
**Testing Framework:** Playwright MCP Tools


#### Playwright MCP Testing Commands:
- Use `mcp_playwright_browser_navigate` to visit the landing page
- Use `mcp_playwright_browser_click` to test button functionality
- Use `mcp_playwright_browser_snapshot` to capture layout states
- Use `mcp_playwright_browser_resize` to test responsive design
- Use `mcp_playwright_browser_wait_for` to test animations and loading states

### Development Priority
1. **Phase 1**: Implement basic layout with working navigation
2. **Phase 2**: Add marquee/ticker functionality
3. **Phase 3**: Implement theme toggle and responsive design
4. **Phase 4**: Add authentication integration
5. **Phase 5**: Comprehensive Playwright testing

### Notes
- Focus on landing page functionality first
- Ensure all interactive elements work before moving to other pages
- Use Playwright MCP tools for automated UI/UX testing
- Maintain neobrutalism aesthetic throughout implementation
